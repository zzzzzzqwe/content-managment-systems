Лабораторная работа №4. Разработка плагина для WordPress
Цель работы
Освоить расширяемую модель данных WordPress: создать CPT (Custom Post Type), пользовательскую таксономию, метаданные с метабоксом в админ-панели, а также реализовать виджет для отображения данных на сайте.

Условие
Соберите учебный плагин USM Notes, который добавляет в сайт раздел «Заметки» с приоритетами и датой напоминания.

Примечание: студент может выбрать свою тему для плагина, но функционал должен быть аналогичным.
Шаг 1. Подготовка среды
В локальной установке WordPress перейдите в папку wp-content/plugins.
Создайте директорию для своего плагина, например usm-notes.
Включите отладку в wp-config.php, установив define('WP_DEBUG', true);.
Шаг 2. Создание основного файла плагина
В папке плагина создайте файл usm-notes.php.
Добавьте в него метаданные плагина (название, описание, версию, автора).
Активируйте плагин в админ-панели WordPress.
Убедитесь, что плагин активен и ошибок нет.
Шаг 3. Регистрация Custom Post Type (CPT)
Добавьте функцию для регистрации CPT «Заметки» (Notes) с помощью register_post_type().
Установите параметры CPT:
публичный (public),
поддержка заголовка, редактора, автора, миниатюры,
наличие архивной страницы,
иконка в админке.
метки (labels) для удобства использования.
Зарегистрируйте CPT при инициализации WordPress с помощью хука init.
Шаг 4. Регистрация пользовательской таксономии
Добавьте функцию для регистрации таксономии «Приоритет» (Priority) с помощью register_taxonomy().
Свяжите таксономию с CPT «Заметки».
Установите параметры таксономии:
иерархическая (как категории),
публичная,
метки (labels) для удобства использования.
Зарегистрируйте таксономию при инициализации WordPress с помощью хука init.
Шаг 5. Добавление метабокса для даты напоминания
Создайте функцию для добавления метабокса в редактор CPT «Заметки» с помощью add_meta_box().
В метабоксе добавьте поле для выбора даты напоминания (используйте HTML5 input type="date").
Создайте функцию для сохранения значения даты при сохранении записи с помощью хука save_post.
Убедитесь, что дата сохраняется корректно и отображается при редактировании записи.
Добавьте проверку nonce для безопасности при сохранении метаданных.
Сделайте поле даты обязательным для заполнения.
Добавьте валидацию даты (например, дата не может быть в прошлом). В случае ошибки сохранения, выведите сообщение об ошибке.
Отобразите дату напоминания в списке записей CPT «Заметки» в админке.
Шаг 6. Создание шорткода для отображения заметок
Создайте функцию для обработки шорткода [usm_notes priority="X" before_date="YYYY-MM-DD"], где priority - фильтр по приоритету, а before_date - фильтр по дате напоминания.
В функции получите и отобразите список заметок, соответствующих фильтрам.
Зарегистрируйте шорткод с помощью add_shortcode().
Добавьте стили для оформления списка заметок.
Обработайте случаи, когда нет заметок, соответствующих фильтрам, и выведите соответствующее сообщение: "Нет заметок с заданными параметрами".
Если priority или before_date не указаны, шорткод должен отображать все заметки.
Шаг 7. Тестирование плагина
Добавьте 5-6 заметок с разными приоритетами и датами напоминания.
Присвойте каждой приоритет (High/Medium/Low) и заполните «Due Date».
Создайте страницу «All Notes» и вставьте шорткод:
[usm_notes] - для отображения всех заметок.
[usm_notes priority="high"] - для отображения заметок с высоким приоритетом.
[usm_notes before_date="2025-04-30"] - для отображения заметок с датой напоминания до 30 апреля 2025 года.
Контрольные вопросы
Чем пользовательская таксономия принципиально отличается от метаполя? Приведи пример, когда выбрать таксономию, а когда - метаданные
Зачем нужен nonce при сохранении метаполей и что произойдёт, если его не проверять?
Какие аргументы register_post_type() и register_taxonomy() чаще всего важны для фронтенда и UX (назови минимум три и объясни почему).
